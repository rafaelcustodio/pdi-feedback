## Codebase Patterns
- Prisma schema at `prisma/schema.prisma`, generated client at `src/generated/prisma/` (gitignored)
- Server Actions pattern: `src/app/(dashboard)/[feature]/actions.ts` for mutations/queries
- Access control: always use `getEffectiveAuth()` from `@/lib/impersonation` (never `auth()` directly)
- Employee form: `src/components/employee-form.tsx` — main form for creating/editing employees
- Profile page: `src/app/(dashboard)/perfil/` — employee self-service view
- Email: `src/lib/email.ts` + `src/lib/email-templates.ts` — Nodemailer with SMTP fallback to console
- Notifications: existing `Notification` model in Prisma schema
- Dark mode: always include `dark:` Tailwind variants
- Timezone: use UTC methods for dates from server (getUTCDate, etc.)
- Prisma generate workaround: `PRISMA_SCHEMA_ENGINE_BINARY=/tmp/schema-engine npx prisma generate` (create mock schema-engine first: `echo '#!/bin/sh' > /tmp/schema-engine && chmod +x /tmp/schema-engine`)
- Migration files must be created manually (no network access to binaries.prisma.sh for schema-engine)
- Pre-existing lint errors (7 errors, 8 warnings) and 8 test failures in sector-schedule-utils.test.ts — not caused by our changes
