## Codebase Patterns
- Prisma generate requires `PRISMA_SCHEMA_ENGINE_BINARY=/dev/null` env var workaround in this environment (no network for engine download)
- Migration SQL must be created manually and applied via `prisma db execute` or direct DB connection since `prisma migrate dev` needs the schema engine binary
- Generated Prisma client files are in `src/generated/prisma/` and have `@ts-nocheck` — they're regenerated via `prisma generate`
- All pre-existing lint errors are in: theme-provider.tsx (setState in effect), programacao/actions.ts, pdi-tracking.tsx, sector-schedule-utils.ts — not from our changes
- Outlook calendar integration: import from `@/lib/microsoft-graph` — getUserToken, createCalendarEvent, updateCalendarEvent, deleteCalendarEvent
- Use Promise.allSettled (never Promise.all) for parallel calendar event creation so failures don't cascade
- Calendar events are fire-and-forget — failures never block the main DB operation
- PDI actions use positional params; Feedback actions use a data object — follow existing patterns when adding fields
- FollowUpsSection has TWO scheduleFollowUp call sites: handleSchedule() and handleScheduleNext()
