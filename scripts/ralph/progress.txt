## Codebase Patterns
- Prisma v7 requires driver adapters (e.g., `PrismaPg`) in PrismaClient constructor - no more `url` in schema.prisma datasource
- Prisma v7 generated client output is at `src/generated/prisma/client` - import from `@/generated/prisma/client`
- Prisma binary engines (binaries.prisma.sh) blocked by firewall - use `PRISMA_SCHEMA_ENGINE_BINARY=/tmp/fake-schema-engine` workaround for CLI commands (validate, generate)
- npm install requires `--no-bin-links` and `--ignore-scripts` flags due to 9p filesystem (WSL2 mount) - then install prisma separately with scripts enabled
- TypeScript: `node node_modules/typescript/bin/tsc --noEmit` (no npx bin links)
- ESLint: `node node_modules/eslint/bin/eslint.js`
- Prisma CLI: `PRISMA_SCHEMA_ENGINE_BINARY=/tmp/fake-schema-engine node node_modules/prisma/build/index.js <command>`
- Project uses Next.js 16 App Router + Tailwind CSS v4 + shadcn/ui (to be installed)
- Portuguese (pt-BR) language for UI labels and content
